# The Shadow Dungeon: An Advanced Text Adventure Engine

Welcome to The Shadow Dungeon, a text-based, choose-your-own-adventure game built in Python. The program drops players into a mysterious underground environment where structural choices dynamically alter the narrative path. Survival depends on the player's capacity to bypass environmental puzzles, evade traps, and decipher magical protective runes under strict time constraints.

---

## Game Loop Architecture & Features

This project serves as a showcase for integrating standalone script components, managing module workspaces, and tracking real-time user metrics within a running terminal application.

### 1. The Left Path: The Orc's Guard Room

Choosing the left door opens up two high-stakes combat/stealth paths:

- **The Stealth Route (Steal):** Bypassing the sleeping Orc requires deploying a chemical sleeping gas trap. The system bridges into the `number_guessing_game` module to launch a variable-difficulty numerical decryption protocol. Succeeding subdues the beast; failing sounds an audible alarm trigger and ends the game loop.
- **The Combat Route (Fight):** Engaging the Orc initiates a fast-paced, 3-round cryptographic encounter. To pierce the enemy's armor defenses, players must parse strings generated by a real-time Caesar Cipher engine.

### 2. The Right Path: The Glittering Vault

Choosing the right door pivots the narrative toward exploration and risk mitigation:

- **The Combination Vault (Chest):** Players discover a treasure chest locked behind a digital frequencies keypad. This node re-utilizes the standalone `number_guessing_game` module to determine entry permissions. Success uncovers the dungeon exit key; failure trips a high-velocity physical spike defensive trap.
- **The Void Well (Well):** A narrative dead-end trap designed to penalize uncalculated exploration.

---

## Technical Highlights

- **Integrated Micro-Modularity:** The program directly calls `chosee_difficulty()` and `play_round()` hooks from an independent `number_guessing_game` file, illustrating decoupled module orchestration.
- **Real-Time Combat Metrics:** The Caesar Cipher combat loop leverages Python's baseline `time.time()` engine to track user response speed down to the millisecond. Taking longer than 10.0 seconds breaks the loop, triggering a failure condition.
- **Procedural Cryptography Engine:** Built-in `encrypt()` handles string shifts mapping precisely to classical $O(N)$ ASCII boundaries via `ord()` and `chr()`, keeping alphanumeric symbols structurally intact while shifting valid case structures.
- **Clean Control Flow:** Isolating environmental nodes into clean functions prevents deeply nested conditional blocks, ensuring optimal memory allocations and execution scaling.

The game splits down distinct computational vectors depending on player decisions:

```text
                           [ start_game() ]
                            /            \
                (left)     /              \  (right)
                          v                v
          [ left_door_room() ]          [ right_door_room() ]
             /            \                /              \
   (steal)  /      (fight) \      (chest) /        (well)  \
           v                v            v                  v
     [ Gas Trap ]   [ fight_orc_room() ] [ Combo Lock ]    [ Tentacle Trap ]
    (Guessing Game)   (Caesar Cipher)   (Guessing Game)     (Game Over)
```
